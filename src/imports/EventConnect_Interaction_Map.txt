# 1. Global Navigation Rules
The bottom navigation bar is persistent on all screens within an event context. It has 5 tabs. The active tab is visually highlighted. Tapping the active tab scrolls to top (does not reload).
| Tab | Tap Action | Target Screen | Badge/Indicator |
| --- | --- | --- | --- |
| Home | Opens Home dashboard | Screen 2: Home | None |
| Participants | Opens Participants with last active tab (default: Match) | Screen 3 or 4: Participants | Badge: count of new matches since last visit |
| QR | Opens QR Code screen (center button, visually prominent) | Screen 5: QR Code | None |
| Chat | Opens Chat list with last active tab (default: All) | Screen 6: Chat List | Badge: total unread message count |
| Profile | Opens own Profile | Screen 8: Profile (own) | Red dot if profile < 100% complete |

| --- | --- | --- | --- |

# 14. Global UI Behavior Rules
## 14.1 Read Receipts
⚠️ BUG FIX: Checkmarks appear ONLY on outgoing (sent) messages. Never on incoming messages.
Single ✓: Message sent to server. Shown on outgoing messages immediately after sending.
Double ✓✓: Message read by recipient. Shown on outgoing messages after recipient opens the chat.
Incoming messages: NO checkmarks. Only show sender avatar, message content, and timestamp.
Chat list preview: Checkmarks shown next to last message ONLY if it was sent by current user.
## 14.2 Match Card Display
⚠️ BUG FIX: Match tab cards must be FULL-SCREEN (one at a time), not a scrollable list.
Match tab: Full-screen card view. One person per screen. Swipe left/right or use buttons.
Liked tab: Scrollable list view with compact cards.
Hidden tab: Scrollable list view with compact cards + "Like" button to restore.
All tab: Scrollable list view with compact cards + action buttons (Profile, Like, Hide, Write now).
## 14.3 Navigation Back Behavior
Back arrow: Always returns to the previous screen in the navigation stack (not a fixed destination).
Bottom tab tap: Goes to root of that tab (e.g., Chat tab → Chat List, not last open conversation).
Swipe from left edge: Same as back arrow (iOS convention). Android uses system back button.
Modal/overlay close: Returns to the screen that opened it. Does not navigate away.
## 14.4 Profile Completion Gate
Required fields: Name, Position, Grade, Company, 7+ Interests.
Blocked features until complete: Match tab shows locked state with message "Complete your profile to unlock matches" and button to edit profile.
Profile widget on Home: Shows completion percentage. Tapping opens edit mode directly.
Unblocked features: All, Liked, Hidden tabs in Participants. Chat. QR. Calendar. Info. Gallery. Network in Real Time.
## 14.5 Empty States
No matches yet: "No matches found. Try completing your profile or adjusting interests."
No chats yet: "No conversations yet. Find someone in Participants and say hi!"
No liked contacts: "You haven't liked anyone yet. Start exploring in Match tab!"
Hidden tab empty: "No hidden contacts."
No free conversations: "No free conversations scheduled yet. Create one from Chat → + Meeting!"
Network in Real Time (before event): "Available during the event. Come back on [event date]!"
Network in Real Time (after event): "This feature was available during the event."
Gallery empty: "No photos yet. Be the first to add one!"
## 14.6 Mutual Match Flow
Trigger: Both users Like each other (via swipe, button, or QR scan).
Notification: Both users receive push: "You and [Name] are a mutual match!"
In-app overlay: "It's a Match!" screen with both profiles. Buttons: "Write now" (opens chat) and "Continue" (dismisses).
Chat list: Conversation moves to Match tab in Chat.
Participants: User appears in Match-filtered views.
